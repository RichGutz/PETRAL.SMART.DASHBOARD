HOY ✓

FUTURO
BAF — Bunker Adjustment Factor: el motor ya tiene la arquitectura para calcular el ajuste cuando el precio del mercado supera el precio base del contrato en más de un umbral (ej. 5%). Esto habilitará la liquidación real de cada barco.

Maestros de Datos · 1 de 6

Tabla: vessels

Define las capacidades de carga y consumos energéticos de cada nave de la flota

Ejemplo Real de Supabase:

{
  "vessel_id": "CONCON_TRADER",
  "vessel_name": "CONCON TRADER",
  "dwt": 19823,
  "vessel_speed": 11.0,
  "vessel_pump_discharge_rate": 500.0,
  "consumption_sea_ifo": 14.0,
  "consumption_port_ifo": 3.5,
  "tce_required": 20000
}

Impacto en simulación: Dicta cuánto tarda en llegar, cuánto consume en el mar (IFO/MDO) y la velocidad máxima a la que puede bombear carga (limitado por el puerto).

Maestros de Datos · 2 de 6

Tabla: routes

Mapea la topología marítima y las distancias entre puertos

Ejemplo Real de Supabase:

{
  "origin_port_id": "ILO",
  "destination_port_id": "MATARANI",
  "route_distance": 69.0,
  "weather_factor": 0.03,
  "description": "Tramo corto de cabotaje sur (Perú)"
}

Impacto en simulación: Provee las millas náuticas necesarias para calcular el tiempo de mar (Sea Days = Distancia / Velocidad Buque). El weather factor agrega un margen de seguridad histórico.

Maestros de Datos · 3 de 6

Tabla: ports

Establece las reglas físicas, los cuellos de botella y límites de operación por muelle

Ejemplo Real de Supabase:

{
  "port_id": "ILO",
  "max_load_rate": 500.0,
  "max_disch_rate": 9999.0,
  "overhead_carga_hrs": 6.0,
  "overhead_descarga_hrs": 6.0
}

Impacto en simulación: Aplica la "Ley del Mínimo". Si un barco descarga a 800 T/h pero el puerto solo acepta 500 T/h, el motor usa 500. Los "overheads" suman horas muertas fijas por fondeo, maniobras y papeleo.

Maestros de Datos · 4 de 6

Tabla: contracts

Define los acuerdos comerciales paraguas entre la Naviera y el Cliente

Ejemplo Real de Supabase:

{
  "contract_id": "SPCC_2025",
  "client_id": "SPCC",
  "origin_port_id": "ILO",
  "destination_port_id": "MATARANI",
  "valid_from": "2025-01-01",
  "valid_to": "2025-12-31"
}

Impacto en simulación: Sirve como validación estructural. Asegura que no se puedan proyectar viajes fuera de vigencia o en rutas no estipuladas. Funciona como llave primaria para buscar las tarifas.

Maestros de Datos · 5 de 6

Tabla: contract_tariffs

Contiene los "Brackets" o escalones de flete dinámico

Ejemplo Real de Supabase:

{
  "contract_id": "SPCC_2025",
  "min_tonnage": 10000.0,
  "max_tonnage": 11500.0,
  "freight_rate": 20.12
}

Impacto en simulación: El motor busca en qué "bracket" cae la cantidad a transportar. Si se transportan 11,000 MT, cobrará $20.12 por tonelada. Esto permite modelar descuentos por volumen.

Maestros de Datos · 6 de 6

Tabla: agency_matrix

Matriz de resolución para los costos fijos portuarios

Ejemplo Real de Supabase:

{
  "port_id": "ILO",
  "operation_type": "CARGA",
  "client_id": "DEFAULT",
  "vessel_id": "DEFAULT",
  "cost": 25500.0
}

Impacto en simulación: El sistema busca siempre la tarifa más específica (Ej: Barco X + Cliente Y). Si no existe una tarifa negociada, hace "fallback" al comodín DEFAULT. Deduce automáticamente los costos base de cada puerto tocado.
